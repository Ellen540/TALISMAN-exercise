I began by running four separate experiments corresponding to positive and negative commands on each actuator channel. By comparing these runs to a baseline run with no actuation, I hoped to identify how each command affected the spot position.
* The first attempt at visualising actuator effects was to represent the response using displacement vectors. The idea was to calculate the overall movement caused by each actuator and display this as an arrow showing both magnitude and direction. I found these results difficult to interrupt, potentially due to the noise skewing the results adn decided to break down the problem into smaller sections focusing on the direction the actuators moved the spot and then by what magnitude. 
* This made it much easier to identify which actuator primarily influenced each coordinate. Np.abs() was useful when comparing displacement magnitudes, as the sign only indicated direction whereas I was interested in the size of the response. 
* Now I had an understanding of the steady-state effects of the actuators, I looked at investigating response speed. The original funcion for logging a run applied a constant command throughout the entire run. I changed this to support time varying commands to measure dynamics. The first approach introudced a command halfway through a run, allowing observations of how the spot responded when the actuator was suddenly activated.
* I attempted to introduce multiple command changes within a single run so that the system would repeatedly move and relax. During this process I encountered issues when trying to return the actuator command to a baseline state. To avoid these problems, I used finite-duration pulses, using actions[t1:t2].
